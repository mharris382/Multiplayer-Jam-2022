Keyword | Description
---|---
`if` | See if/else/elif.(see [[Statements and Control Flow]]
`elif` | See if/else/elif.(see [[Statements and Control Flow]]
`else` | See if/else/elif.(see [[Statements and Control Flow]]
`for` | See for. (see [[Statements and Control Flow]]
`while` | See while (see [[Statements and Control Flow]].
`match` | See [[Match]].
`break` | Exits the execution of the current for or while loop.  (see [[Statements and Control Flow]])
`continue` | Immediately skips to the next iteration of the for or while loop (see [[Statements and Control Flow]])
`pass` | Used where a statement is required syntactically but execution of code is undesired, e.g. in empty functions.
`return` | Returns a value from a function. (see [[Functions]])
`class` | Defines an inner class. (see [[Inner Classes]])
`class_name` | Defines a class name and optional icon for your script. (see [[Inheritence]])
`extends` | Defines what class to extend with the current class. (see [[Inheritence]])
`is` | Tests whether a variable extends a given class, or is of a given built-in type.
`as` | Cast the value to a given type if possible.
`self` | Refers to current class instance.
`tool` | Executes the script in the editor. (see [[Tool Mode]])
`signal` | Defines a signal. (see [[Signals]])
`func` | Defines a function. (see [[Functions]])
`static` | Defines a static function. Static member variables are not allowed. (see [[Static Functions]])
`const` | Defines a constant. (see [[Constants]])
`enum` | Defines an enum. (see [[Enums]])
`var` | Defines a variable. (see [[Data]])
`onready` | Initializes a variable once the Node the script is attached to and its children are part of the scene tree. (see [[onready Keyword]])
`export` | Saves a variable along with the resource it's attached to and makes it visible and modifiable in the editor.
`setget` | Defines setter and getter functions for a variable. (see [[Setter and Getters (aka Properties)]])
`breakpoint` | Editor helper for debugger breakpoints.
`preload` | Preloads a class or variable. See [[Classes as a Resource]].
`yield` | Coroutine support. See [[Coroutines]]
`assert` | Asserts a condition, logs error on failure. Ignored in non-debug builds. See [[Assert]] keyword.
`remote` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`master` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`puppet` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`remotesync` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`mastersync` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`puppetsync` | Networking RPC annotation. See [high-level multiplayer docs](https://docs.godotengine.org/en/stable/tutorials/networking/high_level_multiplayer.html#doc-high-level-multiplayer).
`PI` | PI constant.
`TAU` | TAU constant.
`INF` | Infinity constant. Used for comparisons.
`NAN` | NAN (not a number) constant. Used for comparisons.
